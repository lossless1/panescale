       - Add `stateSave(canvas: CanvasSnapshot): Promise<void>` -- serializes snapshot to JSON string, calls invoke('state_save', { canvas: JSON.stringify(snapshot) })
       - Add `stateLoad(): Promise<CanvasSnapshot | null>` -- calls invoke, parses JSON
       - Define `CanvasSnapshot` type:
         ```typescript
         export interface CanvasSnapshot {
           nodes: SerializedNode[];  // position, size, data (cwd, shellType), zIndex
           viewport: { x: number; y: number; zoom: number };
           maxZIndex: number;
         }
         ```

    2. **`src/lib/persistence.ts`:**
       - `initPersistence()` function that:
         - Subscribes to `canvasStore` changes via `canvasStore.subscribe()`
         - On any state change: debounce 500ms, then call `stateSave(serializeCanvas(state))`
         - Export `forceSave()` that immediately saves (clears any pending debounce)
       - `serializeCanvas(state)`: extracts nodes (position, size, data, zIndex) and viewport into CanvasSnapshot. Strips React Flow internal state that shouldn't be persisted.
       - `deserializeCanvas(snapshot)`: converts CanvasSnapshot back into canvasStore-compatible state

    3. **Update `src/stores/canvasStore.ts`:**
       - In `addTerminalNode`: after adding node, call `forceSave()` (immediate save per PERS-03)
       - In `removeNode`: after removing, call `forceSave()` (immediate save per PERS-03)
       - Add `loadFromDisk()` method: calls `stateLoad()`, deserializes, sets nodes/viewport/maxZIndex
       - Add `hydrated: boolean` flag so the UI can wait for state load before rendering
       - In `loadFromDisk()`, set `restored: true` on each deserialized node's data so TerminalNode can detect it on mount

    4. **Update `src/components/canvas/TerminalNode.tsx`:**
       - Add `restored?: boolean` to `TerminalNodeData` type
       - In the mount `useEffect`, after creating the xterm Terminal and FitAddon, add a restore check: if `data.restored === true` and no live PTY exists yet (ptyId is null), call `pty.spawn(data.cwd, cols, rows, term)` to reconnect the terminal to a new PTY session in the saved working directory. This ensures restored terminals auto-respawn without any user action.

    5. **Update `src/App.tsx`:**
       - On mount: call `canvasStore.loadFromDisk()` to restore persisted state
       - Call `initPersistence()` to start auto-save subscription
       - Show a loading state while `hydrated` is false (brief -- just prevents flash of empty canvas before state loads)
       - Handle app quit: Use Tauri's window close event to force a final save:
         ```typescript
         import { getCurrentWindow } from '@tauri-apps/api/window';
         getCurrentWindow().onCloseRequested(async (event) => {
           await forceSave();
         });
         ```

       - `it.todo('addTerminalNode triggers immediate save')`

    IMPORTANT: Tile create and close must trigger IMMEDIATE save (not debounced) per PERS-03.
