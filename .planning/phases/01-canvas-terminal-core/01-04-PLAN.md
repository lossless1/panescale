  <action>
    Create the frontend bridge to the Rust PTY backend and the focus management system.

    1. **`src/lib/ipc.ts`:** Typed wrappers for PTY Tauri commands:
       ```typescript
       import { invoke, Channel } from '@tauri-apps/api/core';

       export type PtyEvent =
         | { event: 'data'; data: { bytes: number[] } }
         | { event: 'exit'; data: { code: number | null } };

       export async function ptySpawn(cwd: string, cols: number, rows: number, onEvent: Channel<PtyEvent>): Promise<string> {
         return invoke<string>('pty_spawn', { cwd, cols, rows, onEvent });
       }
       export async function ptyWrite(ptyId: string, data: Uint8Array): Promise<void> {
         return invoke('pty_write', { ptyId, data: Array.from(data) });
       }
       export async function ptyResize(ptyId: string, cols: number, rows: number): Promise<void> {
         return invoke('pty_resize', { ptyId, cols, rows });
       }
       export async function ptyKill(ptyId: string): Promise<void> {
         return invoke('pty_kill', { ptyId });
       }
       ```

    2. **`src/hooks/usePty.ts`:** React hook for PTY lifecycle:
       `usePty` manages everything. It returns `{ spawn, write, resize, kill, ptyId, isAlive }`.
       - `spawn(cwd, cols, rows, term: Terminal)`: creates Channel, calls ptySpawn, wires onmessage to term.write (converting number[] to Uint8Array), wires term.onData to ptyWrite. Returns ptyId.
       - `write(data)`: sends to ptyWrite
       - `resize(cols, rows)`: sends to ptyResize
       - `kill()`: sends to ptyKill, cleans up
       - Cleanup on unmount: kill if alive

    3. **`src/hooks/useFocusMode.ts`:** Zustand-based focus mode store + hook:
       ```typescript
       type FocusMode = 'canvas' | 'terminal';
       interface FocusState {
         mode: FocusMode;
         activeTerminalId: string | null;
         enterTerminalMode: (terminalId: string) => void;
         exitToCanvasMode: () => void;
       }
       ```
       - `enterTerminalMode(id)`: sets mode='terminal', activeTerminalId=id
       - `exitToCanvasMode()`: sets mode='canvas', activeTerminalId=null
       - Export both the store and a `useFocusMode()` hook

       Also add global Escape key handler: when mode='terminal' and Escape is pressed, call `exitToCanvasMode()`. This should be in a `useEffect` with capture phase listener, registered in App.tsx or the Canvas component.

    4. **`src/stores/settingsStore.ts`:** Zustand store for terminal settings:
       - `fontFamily: string` (default 'JetBrains Mono, Menlo, Monaco, monospace')
       - `fontSize: number` (default 14)
       - `scrollback: number` (default 1000)
       Per user decision TERM-14: configurable font, font size, and color scheme. Color scheme comes from the theme system (themeStore), not a separate setting for now.

    IMPORTANT: `Channel` import comes from `@tauri-apps/api/core`, NOT from `@tauri-apps/api/event`.
    IMPORTANT: PTY data arrives as `number[]` (JSON-serialized Vec<u8>). Convert to `Uint8Array` before passing to xterm.js `write()`.
    IMPORTANT: Use `term.onData((data) => ptyWrite(ptyId, new TextEncoder().encode(data)))` to wire keyboard input. xterm.js `onData` emits strings.

    2. **`src/components/canvas/TerminalNode.tsx`:**
       - `React.memo` wrapped custom node component
       - `NodeProps` from React Flow, with data type:
         ```typescript
         type TerminalNodeData = {
           cwd: string;
           shellType: string;
         };
         ```
       - Use `usePty()` hook for PTY communication
       - Use `useFocusMode()` for focus state
       - Use `useCanvasStore()` for bringToFront and removeNode
       - Use `useThemeStore()` for terminal theme colors
       - Use settings from `settingsStore` for font/size/scrollback

       **xterm.js setup (in useEffect):**
       - Create `Terminal` with options: fontFamily, fontSize, scrollback from settings, cursorBlink: true, theme from CSS variables (build ITheme from computed CSS vars)
       - Load `FitAddon`
       - Load `WebglAddon` (with try/catch fallback to DOM renderer -- WebGL may fail on some platforms)
       - Call `term.open(containerRef.current)` then `fitAddon.fit()`
       - Call `pty.spawn(data.cwd, cols, rows, term)` to connect to backend
       - Wire `term.onData` -> `pty.write` (keyboard input to PTY)

       **Focus handling:**
       - On click anywhere in the terminal container: call `focusMode.enterTerminalMode(nodeId)`, call `term.focus()`, call `canvasStore.bringToFront(nodeId)`
       - When focus mode changes away from this terminal: call `term.blur()`
       - Visual indicator: when focused (activeTerminalId === nodeId), apply `box-shadow: 0 0 0 2px var(--focus-glow)` or border glow

       **Resize handling:**
       - Use `<NodeResizer>` from @xyflow/react with `minWidth={320}` (~40 cols), `minHeight={200}` (~10 rows)
       - On resize callback: call `fitAddon.fit()` to recalculate cols/rows, then call `pty.resize(term.cols, term.rows)`
       - Use `isVisible={selected}` so handles only show when selected (or always show -- user preference)
       - ResizeObserver on the container div as backup for fit recalculation

       **Close handling:**
       - TitleBar close button calls `pty.kill()` then `canvasStore.removeNode(nodeId)`

       **Copy/paste (TERM-05):**
       - xterm.js has built-in text selection
       - On Cmd/Ctrl+C when terminal has selection: copy to clipboard via `navigator.clipboard.writeText(term.getSelection())`
       - On Cmd/Ctrl+C when no selection: let it pass through to PTY as SIGINT (default xterm behavior)
       - On Cmd/Ctrl+V: read from clipboard, write to PTY via `pty.write(clipboardText)`
       - Wire this in a keydown handler on the terminal container

       **CSS classes on terminal container:**
       - `nodrag` -- prevents drag events from bubbling to React Flow
       - `nowheel` -- prevents scroll from zooming canvas
       - `nopan` -- prevents pan from canvas when interacting with terminal

       **Scroll behavior (per user decision):**
       - Scroll over terminal = scroll terminal output (default xterm behavior + `nowheel` class)
       - Shift+scroll = pan canvas: intercept wheel in the terminal container, if Shift: allow propagation to React Flow. If no Shift: feed to xterm scrollback.
