  <action>
    Build the infinite canvas using @xyflow/react v12.

    1. **`src/stores/canvasStore.ts`:** Zustand store with:
       - `nodes: Node[]` (empty initially)
       - `viewport: Viewport` (`{ x: 0, y: 0, zoom: 1 }`)
       - `maxZIndex: number` (0)
       - `onNodesChange(changes: NodeChange[])` -- applies changes via `applyNodeChanges`
       - `addTerminalNode(position, ptyId, cwd)` -- creates node with type 'terminal', `dragHandle: '.drag-handle'`, default size 640x480, increments maxZIndex
       - `removeNode(id)` -- filters out node
       - `bringToFront(id)` -- increments maxZIndex, sets on node
       - `setViewport(viewport)` -- stores current viewport
       Do NOT add persistence logic yet (Plan 05 handles that). Just the pure canvas state.

    2. **`src/components/canvas/Canvas.tsx`:** ReactFlow wrapper component:
       - Import `@xyflow/react/dist/style.css`
       - Use `useCanvasStore` for nodes and onNodesChange
       - Configure ReactFlow props:
         - `minZoom={0.1}` `maxZoom={2.0}` (10% to 200% per user decision)
         - `panOnScroll={true}` (trackpad two-finger = pan)
         - `panOnDrag={true}` (click+drag on empty space = pan)
         - `zoomOnScroll={false}` (scroll = pan, NOT zoom)
         - `zoomOnPinch={true}` (pinch-to-zoom on trackpad)
         - `selectionOnDrag={false}`
         - `fitView={false}` (start at origin)
         - `nodeTypes` will include 'terminal' (registered but component defined in Plan 04)
       - Handle `onMoveEnd` to store viewport in canvasStore
       - Handle double-click on pane for terminal spawn (will be wired in Plan 04, but register the handler now that calls a no-op or logs)
       - Implement Cmd/Ctrl+scroll = zoom: Use a wheel event handler on the ReactFlow wrapper div. If `e.ctrlKey || e.metaKey`, call `zoomIn()`/`zoomOut()` from `useReactFlow()`. Otherwise let default scroll=pan behavior work.
       - Implement Space+drag: Track Space keydown/keyup in a ref. When Space is held, ensure panOnDrag is active (it already is by default). Add a CSS class to show grab cursor when Space is held.
       - Render two `<Background>` components for layered dot grid (see CanvasBackground or inline).

    3. **Rubber-band effect at zoom limits (LOCKED USER DECISION):**
       Implement via a custom `useOnViewportChange` hook from @xyflow/react:
       - In `onChange` callback: check if `viewport.zoom` hits `minZoom` (0.1) or `maxZoom` (2.0)
       - When at limit: apply a brief CSS transform scale overshoot (e.g., scale to 0.095 or 2.05) on the ReactFlow viewport container, then animate back with a CSS transition (150ms ease-out)
       - This creates a subtle "bounce" feel when the user tries to zoom past limits
       - Use a ref to track if we're at a limit to avoid repeated triggers
       - Apply the transform to the `.react-flow__viewport` element

    4. **`src/components/canvas/CanvasBackground.tsx`:** (Optional -- can be inline in Canvas.tsx)
       Two `<Background>` layers:
       - Minor dots: `variant="dots"`, gap=20, size=1, color=`var(--grid-minor)`
       - Major dots: `variant="dots"`, gap=100, size=2, color=`var(--grid-major)`
       Each with unique `id` prop.

    5. **`src/hooks/useKeyboardShortcuts.ts`:** Global keyboard shortcut handler (capture phase):
       - `Cmd/Ctrl + =` or `Cmd/Ctrl + +`: zoom in (call `zoomIn()` from useReactFlow)
       - `Cmd/Ctrl + -`: zoom out
       - `Cmd/Ctrl + 0`: fit view / zoom to fit all tiles
       - These fire `preventDefault()` + `stopPropagation()` to prevent browser zoom
       - Use `platform.ts` `modKeyCode()` to check the correct modifier per platform
       - Hook takes a `reactFlowInstance` ref or uses `useReactFlow()` internally

    6. **Update `src/App.tsx`:** Render AppShell with `<ReactFlowProvider>` wrapping `<Canvas />` in the main content slot. Import ReactFlowProvider from @xyflow/react.

    Per user decisions: zoom 10%-200%, dot grid with minor+major dots, pan via trackpad scroll + Space+drag + middle-click, Cmd+0 fits all tiles. Rubber-band effect at zoom limits is a LOCKED decision -- implement via useOnViewportChange.

    IMPORTANT: Use `@xyflow/react` imports (NOT `reactflow`). The package was renamed.
    IMPORTANT: `panOnScroll={true}` + `zoomOnScroll={false}` gives the Figma-like scroll=pan behavior.
    IMPORTANT: Middle-click+drag panning is built into React Flow by default when `panOnDrag` includes button 1 (middle). Set `panOnDrag={[0, 1]}` to enable both left-click and middle-click panning on empty canvas.
