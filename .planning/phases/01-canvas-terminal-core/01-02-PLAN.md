       - Validate the shell path exists before returning (on Unix, check with `std::path::Path::exists()`)

    3. **`src-tauri/src/pty/manager.rs`:**
       - Define `PtyEvent` enum (serde-serializable, tagged):
         ```rust
         #[derive(Clone, serde::Serialize)]
         #[serde(rename_all = "camelCase", tag = "event", content = "data")]
         pub enum PtyEvent {
             Data { bytes: Vec<u8> },
             Exit { code: Option<u32> },
         }
         ```
       - Define `PtySession` struct holding:
         - `writer: Box<dyn Write + Send>` (master writer clone)
         - `child: Box<dyn portable_pty::Child + Send + Sync>`
         - `reader_handle: Option<std::thread::JoinHandle<()>>`
       - Define `PtyManager` struct:
         - `sessions: Arc<Mutex<HashMap<String, PtySession>>>`
       - Implement `PtyManager::new() -> Self`
       - Implement `PtyManager::spawn(id, cwd, cols, rows, channel: Channel<PtyEvent>) -> Result<()>`:
         - Call `detect_default_shell()` to get the shell
         - Create PTY via `native_pty_system().openpty(PtySize { rows, cols, pixel_width: 0, pixel_height: 0 })`
         - Build command with `CommandBuilder::new(&shell)`, set `cmd.cwd(&cwd)`, set `cmd.env("TERM", "xterm-256color")`
         - Spawn child via `pair.slave.spawn_command(cmd)`
         - Drop `pair.slave` after spawn (important for proper EOF detection)
         - Clone master reader via `pair.master.try_clone_reader()`
         - Spawn a **dedicated OS thread** (`std::thread::spawn`, NOT tokio) for reading:
           - Read in 4KB buffer loop
           - On `Ok(0)` or `Err(_)`: send `PtyEvent::Exit`, break
           - On `Ok(n)`: send `PtyEvent::Data { bytes: buf[..n].to_vec() }`
         - Clone master writer via `pair.master.try_clone_writer()`
         - Store session in HashMap
       - Implement `PtyManager::write(id, data: &[u8]) -> Result<()>`:
         - Lock sessions, get writer, write bytes
       - Implement `PtyManager::resize(id, cols, rows) -> Result<()>`:
         - Lock sessions, call `master.resize(PtySize { ... })`
         - IMPORTANT: The master handle for resize must be stored. Store `pair.master` in PtySession alongside writer.
         - Actually revise: PtySession should store `master: Box<dyn MasterPty + Send>` for resize, and `writer: Box<dyn Write + Send>` for writes. The reader is cloned before storing.
       - Implement `PtyManager::kill(id) -> Result<()>`:
         - Remove session from HashMap
         - Kill child process
         - Drop master (closes PTY)
         - Optionally join reader thread (with timeout to avoid blocking)

    IMPORTANT: Use `std::thread::spawn` for the PTY reader, NOT `tokio::spawn`. PTY reads are blocking I/O and will starve the tokio runtime if run on the async pool.
    IMPORTANT: Drop the slave end after spawning -- failure to do so prevents EOF detection.
    IMPORTANT: Store both `master` (for resize) and a cloned `writer` (for input). portable-pty's `try_clone_writer()` returns a `Box<dyn Write + Send>`.

    Create Tauri command wrappers and register them with the app.

    1. **`src-tauri/src/pty/commands.rs`:** Four async Tauri commands:

       ```rust
       #[tauri::command]
       pub async fn pty_spawn(
           cwd: String,
           cols: u16,
           rows: u16,
           on_event: Channel<PtyEvent>,
           state: tauri::State<'_, PtyManager>,
       ) -> Result<String, String> {
           let id = uuid::Uuid::new_v4().to_string();
           state.spawn(id.clone(), cwd, cols, rows, on_event)
               .map_err(|e| e.to_string())?;
           Ok(id)
       }
       ```

       - `pty_spawn(cwd, cols, rows, on_event: Channel<PtyEvent>) -> Result<String>` -- returns pty_id
       - `pty_write(pty_id: String, data: Vec<u8>) -> Result<()>` -- writes bytes to PTY
       - `pty_resize(pty_id: String, cols: u16, rows: u16) -> Result<()>` -- resizes PTY
       - `pty_kill(pty_id: String) -> Result<()>` -- kills PTY session

       All commands access `PtyManager` via `tauri::State<'_, PtyManager>`.
       All commands return `Result<T, String>` (Tauri requires String errors).
