1. Create `src/stores/projectStore.ts` with Zustand store:
   ```typescript
   interface Project {
     path: string;
     name: string; // basename of path
   }

   interface ProjectState {
     projects: Project[];
     activeProjectIndex: number;
     activeProject: Project | null; // derived getter
     viewMode: 'tree' | 'feed';    // per-project view mode toggle
     openProject: (path: string) => void;
     closeProject: (path: string) => void;
     setActiveProject: (index: number) => void;
     setViewMode: (mode: 'tree' | 'feed') => void;
   }
   ```
   - `openProject(path)`: Adds project to `projects` array if not already present, sets it as active. Extracts `name` from path basename.
   - `closeProject(path)`: Removes from array, adjusts `activeProjectIndex` if needed.
   - `setActiveProject(index)`: Sets `activeProjectIndex`.
   - `setViewMode(mode)`: Sets `viewMode`.
   - Persist to localStorage via zustand persist middleware (key: "excalicode-projects").
   - Expose a computed `activeProject` that returns `projects[activeProjectIndex]` or null.

4. Create `src/components/sidebar/FileTree.tsx`:
   - Uses `useProjectStore` to get active project path
   - Maintains `expandedDirs: Set<string>` state and `dirContents: Map<string, FileEntry[]>` cache
   - On mount / active project change: loads root directory via `fsReadDir(activeProject.path)`

   - Shows "Open Folder" button when no project is open

   - If multiple projects open, show project selector dropdown at top (project name, click to switch)

   - Each file row: file name, path relative to project root (truncated), modified date formatted as relative time (e.g., "2 hours ago")

   - Clicking a file row: emits an `onFileSelect` callback (same as in FileTree -- passed down from Sidebar)
