  <files>src-tauri/icons/icon.svg, src-tauri/icons/icon.png, src-tauri/icons/32x32.png, src-tauri/icons/128x128.png, src-tauri/icons/128x128@2x.png, src-tauri/icons/icon.icns, src-tauri/icons/icon.ico</files>
  <read_first>src-tauri/tauri.conf.json</read_first>
  <action>
1. Create `src-tauri/icons/icon.svg` with a gradient abstract icon design:
   - Shape: Overlapping rounded rectangles/panels suggesting panes/windows on a canvas
   - Gradient: Use a vibrant gradient -- suggest deep purple (#7C3AED) to electric blue (#3B82F6) to teal (#06B6D4) sweep
   - The shape should be distinctive at small sizes (32x32) -- avoid fine detail that disappears
   - Add subtle depth: one panel slightly overlapping another with slight opacity or offset
   - Rounded corners (rx/ry) for modern feel
   - No text in the icon
   - ViewBox: 0 0 512 512

2. Convert SVG to PNG at required sizes using `sips` (macOS built-in):
   - First, render SVG to a high-res PNG. Since sips cannot convert SVG directly, use `qlmanage -t -s 1024 -o /tmp/ icon.svg` or write a quick node script using sharp/canvas. Alternatively, use `rsvg-convert` if available, or install it via `brew install librsvg`.
   - Simplest approach: Use a node script with the `sharp` package (already available or install it):
     ```
     npm install --save-dev sharp
     ```
     Then create a temp script that:
     a. Reads icon.svg
     b. Outputs: icon.png (512x512), 32x32.png, 128x128.png, 128x128@2x.png (256x256)
   - For icon.ico: Use sharp to create 256x256 PNG, then use `png2ico` or write ICO manually. Alternatively use the `to-ico` npm package or `sharp` with ico output.
   - For icon.icns: Use `iconutil` (macOS built-in):
     a. Create iconset directory with required sizes (16x16, 32x32, 128x128, 256x256, 512x512 + @2x variants)
     b. Run `iconutil -c icns icon.iconset -o icon.icns`

   Concrete steps:
   a. `npm install --save-dev sharp to-ico`
   b. Write `scripts/generate-icons.mjs` that reads icon.svg via sharp, outputs all sizes
   c. Run the script
   d. Verify all icon files exist and have correct dimensions
   e. Keep the script for future regeneration

3. Verify icon files:
   - `sips -g pixelHeight -g pixelWidth src-tauri/icons/32x32.png` should show 32x32
   - `sips -g pixelHeight -g pixelWidth src-tauri/icons/128x128.png` should show 128x128
   - `sips -g pixelHeight -g pixelWidth src-tauri/icons/128x128@2x.png` should show 256x256
   - `file src-tauri/icons/icon.icns` should show "Apple Icon Image"
   - `file src-tauri/icons/icon.ico` should show ICO format
  </action>
