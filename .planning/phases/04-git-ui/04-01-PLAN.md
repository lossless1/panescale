**Types (all derive serde::Serialize):**
- `GitStatusEntry { path: String, status: String }` — status is one of: "staged_new", "staged_modified", "staged_deleted", "staged_renamed", "modified", "deleted", "renamed", "untracked", "conflicted"
- `DiffHunk { header: String, old_start: u32, new_start: u32, old_lines: u32, new_lines: u32, lines: Vec<DiffLine> }`
- `DiffLine { origin: char, content: String, old_lineno: Option<u32>, new_lineno: Option<u32> }`
- `GitFileDiff { path: String, hunks: Vec<DiffHunk> }`
- `GitBranch { name: String, is_current: bool, is_remote: bool }`
- `GitCommitInfo { oid: String, short_oid: String, message: String, author: String, author_email: String, timestamp: i64, parent_oids: Vec<String>, files_changed: Vec<String> }`
- `GitStashEntry { index: usize, message: String, oid: String }`
- `GitConflictEntry { path: String, has_ours: bool, has_theirs: bool, has_ancestor: bool }`

**Commands:**
- `git_status(repo_path: String) -> Result<Vec<GitStatusEntry>, String>` — Use `Repository::statuses()` with `StatusOptions` (include_untracked, recurse_untracked_dirs, include_ignored=false). Map status flags to entry categories. A file can appear in BOTH staged and unstaged if it has index + workdir changes.
- `git_stage_file(repo_path, file_path) -> Result<(), String>` — `index.add_path()` then `index.write()`. CRITICAL: must call `index.write()` to persist.
- `git_unstage_file(repo_path, file_path) -> Result<(), String>` — If HEAD exists, use `repo.reset_default(Some(&tree.into_object()), [&file_path])`. If no HEAD (initial commit), use `index.remove_path()` then `index.write()`. Do NOT use `index.remove_path` for files that exist in HEAD (it would delete them from tracking).
- `git_commit(repo_path, message) -> Result<String, String>` — Get signature via `repo.signature()` with fallback to `Signature::now("Unknown", "unknown@local")`. Write tree from index, get HEAD parent (empty vec for initial commit). Return OID string.
- `git_diff_file(repo_path, file_path, staged: bool) -> Result<GitFileDiff, String>` — If staged=true, diff HEAD tree to index (`diff_tree_to_index`). If staged=false, diff index to workdir (`diff_index_to_workdir`). Use `DiffOptions` with pathspec filter. Collect hunks and lines via `Diff::foreach()` callback. Each line origin is '+', '-', or ' '.
- `git_stage_hunk(repo_path, file_path, hunk_index: usize) -> Result<(), String>` — Get the diff for the file (index to workdir), extract the specific hunk using `Patch::from_diff()`, build a patch buffer containing ONLY that hunk (with correct diff header and hunk header), create `Diff::from_buffer()`, then `repo.apply(&diff, ApplyLocation::Index, None)`.
- `git_unstage_hunk(repo_path, file_path, hunk_index: usize) -> Result<(), String>` — Similar to stage_hunk but diff HEAD tree to index for staged changes, and apply with `ApplyLocation::Index` in reverse (negate the patch by swapping +/- lines and old/new counts).
- `git_branches(repo_path) -> Result<Vec<GitBranch>, String>` — `repo.branches(None)` iterator. For each, get name, check `is_head()` for current, `branch.get().is_remote()` for remote.
- `git_create_branch(repo_path, name: String) -> Result<(), String>` — `repo.branch(&name, &head_commit, false)`.
- `git_switch_branch(repo_path, name: String) -> Result<(), String>` — `repo.set_head(&format!("refs/heads/{}", name))` then `repo.checkout_head(Some(&mut CheckoutBuilder::default().force()))`. CRITICAL: must checkout_head after set_head to update working directory.
- `git_delete_branch(repo_path, name: String) -> Result<(), String>` — Find branch, call `branch.delete()`. Refuse to delete current branch.
- `git_log(repo_path, limit: usize, skip: usize) -> Result<Vec<GitCommitInfo>, String>` — `repo.revwalk()` with TOPOLOGICAL | TIME sort, push_head. Skip/take for pagination. For each commit, populate `files_changed` by diffing the commit tree against its first parent tree using `repo.diff_tree_to_tree(parent_tree, commit_tree, None)` and collecting delta paths via `diff.deltas()`. For root commits (no parent), diff against an empty tree. Return both `parent_oids` (for graph rendering) and `files_changed`.
- `git_stash_save(repo_path, message: String) -> Result<(), String>` — `repo.stash_save()` with signature and message.
- `git_stash_list(repo_path) -> Result<Vec<GitStashEntry>, String>` — `repo.stash_foreach()` collecting entries.
- `git_stash_apply(repo_path, index: usize) -> Result<(), String>` — `repo.stash_apply(index, None)`.
- `git_stash_pop(repo_path, index: usize) -> Result<(), String>` — `repo.stash_pop(index, None)`.
- `git_stash_drop(repo_path, index: usize) -> Result<(), String>` — `repo.stash_drop(index)`.
- `git_conflicts(repo_path) -> Result<Vec<GitConflictEntry>, String>` — `repo.index()?.conflicts()` iterator, collect path and which sides exist (ours/theirs/ancestor).
- `git_resolve_conflict(repo_path, file_path, resolution: String) -> Result<(), String>` — resolution is "ours" or "theirs". For "ours": checkout ours version via `CheckoutBuilder::default().force().path(&file_path)` with `repo.checkout_index()`. For "theirs": similar but using their entry. Then `index.add_path()` and `index.write()` to mark resolved.
- `git_is_repo(repo_path: String) -> Result<bool, String>` — Try `Repository::open()`, return Ok(true) or Ok(false).
