1. **Create `src/components/sidebar/git/StashSection.tsx`:**
- Props: `{ repoPath: string }`
- Reads `stashes` from `useGitStore`.
- Collapsible section header "Stashes" with count badge.
- **Stash save**: Input field + "Stash" button at top of section. On click calls `gitStashSave(repoPath, message)` then `refreshStashes(repoPath)` and `refresh(repoPath)` (status changes after stash). Default message if empty: "WIP on {currentBranch}".
- **Stash list**: Each stash item shows: index label ("stash@{0}"), message (truncated to 1 line), short OID.
- **Stash actions**: Each stash item has three hover-visible buttons:
  - "Apply" (keeps stash): calls `gitStashApply(repoPath, entry.index)`.
  - "Pop" (apply + remove): calls `gitStashPop(repoPath, entry.index)`.
  - "Drop" (discard): calls `gitStashDrop(repoPath, entry.index)` with confirmation dialog (window.confirm).
- After any stash action, refresh both stashes and status. IMPORTANT: stash indices shift after drop/pop, so always refresh the full list.
- Style: fontSize 12, match other sidebar sections. Stash message in italic.

2. **Create `src/components/sidebar/git/ConflictSection.tsx`:**
- Props: `{ repoPath: string }`
- Reads `conflicts` from `useGitStore`.
- Only renders when conflicts array is non-empty. Shows a warning-colored header: "Merge Conflicts" with red/orange accent.
- For each conflicted file:
  - File path (basename highlighted).
  - Three action buttons:
    - "Accept Ours": calls `gitResolveConflict(repoPath, entry.path, "ours")`.
    - "Accept Theirs": calls `gitResolveConflict(repoPath, entry.path, "theirs")`.
    - "Open in Editor": not functional yet (shows a tooltip "Open in terminal editor" -- this is a v2 feature). For now, just show the button disabled or as a placeholder.
  - Indicators for which sides exist: show "ours" checkmark if has_ours, "theirs" checkmark if has_theirs.
- After resolving a conflict, refresh conflicts and status.
- When all conflicts are resolved (conflicts array becomes empty), the section auto-hides and shows a brief "All conflicts resolved" success message.
- Style: Warning/error styling with orange/red border on the section. File items match FileChangeItem style.
