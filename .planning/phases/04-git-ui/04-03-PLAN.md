1. **Create `src/components/sidebar/git/CommitGraph.tsx`:**
- Implements the lane assignment algorithm from the research:

```typescript
interface CommitNode {
  oid: string;
  parentOids: string[];
  column: number;
  row: number;
  color: string;
}
```

- `assignLanes(commits: GitCommitInfo[]): CommitNode[]` — Algorithm:
  - Maintain a `lanes: (string | null)[]` array tracking which OID occupies each column.
  - For each commit: find its lane (where its OID was reserved), or assign next available.
  - Free the lane, then reserve lanes for parents (first parent stays in same column, merge parents get new columns).
  - Colors: cycle through `["#4fc3f7", "#81c784", "#ffb74d", "#f06292", "#ba68c8", "#4db6ac"]`.

- `CommitGraph` component:
  - Props: `{ nodes: CommitNode[], rowHeight: number }` (rowHeight matches the commit item height).
  - Renders an SVG element positioned to the left of the commit list.
  - For each node: draw a circle at (column * columnWidth + offset, row * rowHeight + rowHeight/2).
  - For connections: draw vertical lines (same column parent) or Bezier curves (cross-column merge).
  - Vertical line: straight SVG line between commit and parent in same column.
  - Merge curve: `<path>` with cubic Bezier from commit position to parent position across columns.
  - Column width: 12px. Circle radius: 4px. Line stroke width: 2px. Colors match commit node colors.
