```typescript
openRemoteProject: (remotePath: string, sshSessionId: string, sshHost: string) => {
  const { projects } = get();
  const displayPath = `${sshHost}:${remotePath}`;
  const existingIndex = projects.findIndex((p) => p.path === displayPath);
  if (existingIndex >= 0) {
    set({ activeProjectIndex: existingIndex });
    return;
  }
  const newProject: Project = {
    path: displayPath,
    name: basename(remotePath),
    isRemote: true,
    sshSessionId,
    sshHost,
  };
  set({
    projects: [...projects, newProject],
    activeProjectIndex: projects.length,
  });
},
```

**1. Create RemoteFileTree.tsx** at `src/components/sidebar/RemoteFileTree.tsx`.

This component mirrors the local FileTree pattern but uses `sshReadRemoteDir` IPC instead of `fsReadDir`.

Props: none (reads activeProject from projectStore)

State:
- `expandedDirs: Set<string>` -- track expanded remote directories
- `dirContents: Map<string, RemoteFileEntry[]>` -- cache remote directory contents
- `loading: boolean` -- initial load state
- `error: string | null` -- error state

Behavior:
- On mount / when activeProject changes, if `activeProject.isRemote && activeProject.sshSessionId`:
  - Set `loading: true`
  - Call `sshReadRemoteDir(activeProject.sshSessionId, remotePath)` where `remotePath` is extracted from `activeProject.path` (the part after the colon in `user@host:/path`)
  - Set root entries into `dirContents`
  - Set `loading: false`
  - On error: set `error` state with message
- `toggleDir(path)`: If expanded, collapse. If collapsed, call `sshReadRemoteDir` and add entries to `dirContents`, then expand.

Rendering structure:
```jsx
<div style={{ flex: 1, overflow: "auto" }}>
  {/* Remote project header bar */}
  <div style={{
    padding: "4px 8px",
    fontSize: 10,
    color: "var(--text-secondary)",
    display: "flex",
    alignItems: "center",
    justifyContent: "space-between",
    borderBottom: "1px solid var(--border)",
  }}>
    <span>Remote: {activeProject.sshHost}:{remotePath}</span>
    <button
      onClick={handleDisconnect}
      title={`Disconnect from ${activeProject.sshHost}`}
      style={{ background: "none", border: "none", color: "var(--text-secondary)", cursor: "pointer", padding: "2px 4px", fontSize: 12 }}
    >
      &#x2715;
    </button>
  </div>

  {/* 3px left accent border per UI-SPEC */}
  <div style={{ borderLeft: "3px solid rgba(var(--accent-rgb, 99, 102, 241), 0.4)", minHeight: "100%" }}>
    {loading && (
      <div style={{ padding: 16, textAlign: "center", fontSize: 12, color: "var(--text-secondary)" }}>
        <span style={{ display: "inline-block", animation: "spin 800ms linear infinite" }}>&#x21BB;</span>
        {" "}Loading remote files...
      </div>
    )}
    {error && (
      <div style={{ padding: 16, fontSize: 12, color: "#ef4444" }}>
        {error}{" "}
        <span onClick={handleRetry} style={{ cursor: "pointer", textDecoration: "underline" }}>Reconnect?</span>
      </div>
    )}
    {!loading && !error && renderEntries(rootEntries, 0)}
  </div>
</div>
```

For the left accent border: since CSS custom properties can't be decomposed into RGB components easily with inline styles, use a literal color value that matches var(--accent) at 40% opacity. The app's accent color is approximately `#6366f1` (indigo). Use `borderLeft: "3px solid rgba(99, 102, 241, 0.4)"`.

`renderEntries(entries, depth)`: Recursive function rendering each RemoteFileEntry:
- Directories: clickable row with expand/collapse chevron, click toggles expansion, indent by `depth * 16 + 8` px.
- Files: row with file icon, same indent. Read-only (no click action for remote files in this phase).
- Style matches local FileTree: 12px font, var(--text-primary), padding 2px 0, hover background var(--bg-secondary).
- Empty directory: italic "Empty directory" text in var(--text-secondary).

`handleDisconnect`: Calls `sshDisconnect(activeProject.sshSessionId)` from ipc.ts, then `closeProject(activeProject.path)` from projectStore.

`handleRetry`: Reloads the root directory by calling `sshReadRemoteDir` again.

Add a CSS keyframe for the spinner. Since we're using inline styles, add a `<style>` tag inside the component:
```jsx
<style>{`@keyframes spin { from { transform: rotate(0deg); } to { transform: rotate(360deg); } }`}</style>
```
