  <action>
Create `.github/workflows/release.yml` with the following structure:

**Trigger:** `on: push: tags: ['v*.*.*']`

**Permissions:** `contents: write` (needed to create GitHub Releases)

**Jobs — three platform builds running in parallel + one publish job:**

### Job 1: `build-macos` (runs-on: macos-latest)
Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: 20` and `cache: 'npm'`
3. `dtolnay/rust-toolchain@stable` with `targets: aarch64-apple-darwin,x86_64-apple-darwin`
4. Cache Rust: `actions/cache@v4` with path `~/.cargo/registry, ~/.cargo/git, src-tauri/target` and key based on `Cargo.lock`
5. `npm ci`
6. `tauri-apps/tauri-action@v0` with:
   - `tagName: ${{ github.ref_name }}`
   - `releaseName: 'Panescale ${{ github.ref_name }}'`
   - `releaseBody: ''` (auto-generated notes added by publish job)
   - `releaseDraft: true`
   - `prerelease: false`
   - `includeUpdaterJson: true`
   - `args: --target universal-apple-darwin` (produces universal binary .dmg)
   - env: `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`
   - No signing env vars (unsigned per user decision)

### Job 2: `build-linux` (runs-on: ubuntu-22.04)
Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: 20` and `cache: 'npm'`
3. `dtolnay/rust-toolchain@stable`
4. Install system deps: `sudo apt-get update && sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf libgtk-3-dev`
5. Cache Rust (same pattern as macOS)
6. `npm ci`
7. `tauri-apps/tauri-action@v0` with:
   - Same tag/release config as macOS
   - `includeUpdaterJson: true`
   - No `args` needed (defaults to host target, produces AppImage + deb)
   - env: `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`

### Job 3: `build-windows` (runs-on: windows-latest)
Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: 20` and `cache: 'npm'`
3. `dtolnay/rust-toolchain@stable`
4. Cache Rust (same pattern, Windows paths: `~/.cargo/registry, ~/.cargo/git, src-tauri/target`)
5. `npm ci`
6. `tauri-apps/tauri-action@v0` with:
   - Same tag/release config as macOS
   - `includeUpdaterJson: true`
   - No `args` needed (produces MSI + NSIS)
   - env: `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`
   - No signing env vars (unsigned per user decision)

### Job 4: `publish` (runs-on: ubuntu-latest, needs: [build-macos, build-linux, build-windows])
Steps:
1. Use `gh` CLI to update the draft release: convert from draft to published, add auto-generated release notes
2. `gh release edit ${{ github.ref_name }} --draft=false --generate-notes --repo ${{ github.repository }}`
3. env: `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`

**Important details:**
- The `tauri-apps/tauri-action` automatically creates a draft release and uploads artifacts. Using `releaseDraft: true` lets all three platform jobs upload to the same draft release before publishing.
- `includeUpdaterJson: true` generates the `latest.json` manifest that the Tauri updater endpoint consumes.
- No code signing secrets are referenced (per user decision: unsigned builds in v1). Add comments like `# Future: Add APPLE_CERTIFICATE, APPLE_CERTIFICATE_PASSWORD, etc. for signing` as placeholders.
- Rust cache key should include a hash of `src-tauri/Cargo.lock`.
