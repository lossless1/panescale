<objective>
Fix the broken workspace-rename flow. User reports rename is not working.
</objective>

<root-cause>
Three interacting bugs in the workspaces dropdown rows:

1. **Nested interactive elements.** Each row was a `<button>` with the rename pencil `<span>` and delete `<span>` nested inside. Clicking the pencil relied solely on React's synthetic `stopPropagation` in the click handler — but the parent button's handler (`switchWorkspace + setShowWorkspaces(false)`) could still fire in some event orderings, closing the dropdown before `window.prompt` returned.

2. **Double-click rename shortcut was broken.** A dblclick fires two single-click events first. The first click ran `switchWorkspace + setShowWorkspaces(false)`, unmounting the row before the dblclick event could land — so double-click rename never fired.

3. **Missing `onMouseDown` stopPropagation** on the pencil button meant the outside-click detector (which listens to `mousedown`) could compete with the rename action.
</root-cause>

<tasks>
- Convert each workspace row from `<button>` to `<div role="button">` (valid HTML, no nesting warnings).
- Remove the broken `onDoubleClick` rename shortcut — the pencil button is the only rename path.
- Add `onMouseDown={(e) => e.stopPropagation()}` to the pencil and delete spans so the outer row's click handler and the outside-click detector stay out of the way.
- Trim + empty-check the prompt result before calling `renameWorkspace`.
- Harden `renameWorkspace` in `workspacesStore.ts` to call `persistActiveSnapshot()` — this merges the rename atomically with current canvas/projects state, avoiding a TOCTOU race where an in-flight save could overwrite the rename.
</tasks>
