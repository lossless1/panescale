Modify NoteNode to use shiki for code block highlighting within markdown:

1. Import the shared highlighter from FilePreviewNode pattern:
   - Import `{ createHighlighter, type Highlighter }` from "shiki"
   - Create module-level `highlighterPromise` and `loadedLangs` Set (same pattern as FilePreviewNode)
   - Create `getHighlighter(lang: string)` function that lazily loads languages

2. Create custom marked renderer for code blocks:
   ```typescript
   import { Renderer } from "marked";

   const createCodeRenderer = (highlighter: Highlighter | null): Renderer => {
     const renderer = new Renderer();
     renderer.code = ({ text, lang }: { text: string; lang?: string }) => {
       const language = lang || "text";
       if (highlighter && loadedLangs.has(language)) {
         return highlighter.codeToHtml(text, { lang: language, theme: "one-dark-pro" });
       }
       // Fallback to plain code block
       return `<pre><code class="language-${language}">${text.replace(/</g, '&lt;').replace(/>/g, '&gt;')}</code></pre>`;
     };
     return renderer;
   };
   ```

3. In the component:
   - Add `const [highlighter, setHighlighter] = useState<Highlighter | null>(null)`
   - Add useEffect to load highlighter on mount (load common langs: typescript, javascript, python, rust, bash, json)
   - Update the markdown rendering useEffect to:
     a. Extract code block languages from markdownContent using regex `/```(\w+)/g`
     b. Ensure those languages are loaded via getHighlighter
     c. Call `marked.parse(markdownContent, { renderer: createCodeRenderer(highlighter), async: false })`

4. Apply `.prose` class to the markdown preview div (replace current `note-markdown-preview` class or add alongside)

5. Keep the theme-aware styling: use "one-dark-pro" for dark theme, "github-light" for light theme (read from themeStore if available, or default to one-dark-pro)
