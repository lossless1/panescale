2. Create `e2e/tmux/helpers.ts` with utilities:
   - `TMUX_SOCK` constant: path.join(os.homedir(), 'Library/Application Support/panescale/tmux.sock')
   - `listSessions()`: execSync `tmux -S "$SOCK" list-sessions -F "#{session_name}" 2>/dev/null || echo ""`, parse into string[], filter to `exc-` prefix
   - `sessionCount()`: listSessions().length
   - `hasSession(name: string)`: boolean check via `tmux -S "$SOCK" has-session -t name`
   - `capturePane(sessionName: string)`: execSync `tmux -S "$SOCK" capture-pane -t name -p`, return string
   - `sendKeys(sessionName: string, keys: string)`: execSync `tmux -S "$SOCK" send-keys -t name "keys" Enter`
   - `getOption(option: string)`: execSync `tmux -S "$SOCK" show-options -g option`, return string
   - `killServer()`: execSync `tmux -S "$SOCK" kill-server 2>/dev/null || true` (cleanup)
   - `waitForSession(predicate, timeoutMs = 10000)`: poll with 500ms interval until predicate returns true or timeout, throw on timeout

3. Add npm script to `package.json`:
   - `"test:e2e:integration": "playwright test --config playwright.integration.config.ts"`

Create `e2e/tmux/tmux-integration.spec.ts` with 4 test scenarios. Import helpers from `./helpers`.

**Setup (beforeEach):**
- Call `killServer()` to clean orphan sessions from previous runs
- Navigate to baseURL (localhost:1420)
- Wait for the React Flow canvas to render (`.react-flow` selector)

**Teardown (afterEach):**
- Call `killServer()` to clean up

**Test 1: "terminal spawns with working shell"**
- Double-click on the canvas to spawn a terminal node (use `page.dblclick('.react-flow__pane')`)
- Wait for `.xterm` element to appear inside a `.react-flow__node` (page.waitForSelector, timeout 10s)
- Use `waitForSession(() => sessionCount() >= 1, 10000)` to wait for tmux session
- Get the session name from `listSessions()[0]`
- Use `sendKeys(session, 'echo e2e-test-marker')` then sleep 500ms
- Use `capturePane(session)` and assert output contains 'e2e-test-marker'

**Test 2: "only 1 tmux session per terminal tile (no double-spawn)"**
- Double-click canvas to spawn terminal
- Wait for session to appear via `waitForSession`
- Wait an additional 3 seconds (to let React StrictMode double-mount settle)
- Assert `sessionCount() === 1` -- exactly one session, not two

**Test 3: "no tmux status bar visible"**
- Double-click canvas to spawn terminal
- Wait for session to appear
- Call `getOption('status')` and assert it contains 'off'

**Test 4: "terminal session persists after app close and reopen"**
- Double-click canvas to spawn terminal
- Wait for session and capture the session name
- Close the page (`page.close()`) -- simulates app close from frontend side
- Assert `hasSession(sessionName)` is true -- tmux session survives because tmux server is independent
- NOTE: Full app restart test (verifying reattach after `cargo tauri dev` restart) requires stopping/starting the Tauri process which is beyond Playwright's scope for this initial setup. Add a comment documenting this limitation and suggesting a future shell script wrapper.

**Important notes in action blocks:**
- All tmux commands use the dedicated Panescale socket, NOT the user's default tmux
- Tests assume `cargo tauri dev` is already running (no webServer auto-start)
- Add a descriptive comment at the top of the file explaining how to run: "Start the app with `cargo tauri dev` first, then run `npm run test:e2e:integration`"
