In NoteNode.tsx, intercept clicks on rendered markdown links to spawn WebView tiles instead of opening the system browser:

1. Import `addWebViewNode` from canvasStore: `const addWebViewNode = useCanvasStore((s) => s.addWebViewNode);`

2. Add a click handler on the markdown preview div (the one with `dangerouslySetInnerHTML`). Attach an `onClick` handler:
   ```typescript
   const handlePreviewClick = useCallback((e: React.MouseEvent<HTMLDivElement>) => {
     const anchor = (e.target as HTMLElement).closest('a');
     if (!anchor) return;
     const href = anchor.getAttribute('href');
     if (!href) return;

     // Only intercept http/https links (not anchors or relative paths)
     if (!href.startsWith('http://') && !href.startsWith('https://')) return;

     e.preventDefault();
     e.stopPropagation();

     // Get the NoteNode's position from the canvas store to place WebView nearby
     const nodes = useCanvasStore.getState().nodes;
     const thisNode = nodes.find(n => n.id === id);
     const nodeWidth = (thisNode?.style?.width as number) ?? 300;
     const position = {
       x: (thisNode?.position.x ?? 0) + nodeWidth + 20,
       y: thisNode?.position.y ?? 0,
     };

     addWebViewNode(position, href);
   }, [id, addWebViewNode]);
   ```

3. Apply the handler to the preview div:
   ```tsx
   <div
     className="prose note-markdown-preview"
     dangerouslySetInnerHTML={{ __html: renderedHtml }}
     onClick={handlePreviewClick}
     style={{ ... }}
   />
   ```

4. Remove or guard any existing `target="_blank"` or `window.open` behavior from the marked renderer if present. The default marked renderer creates `<a href="...">` tags without target="_blank", so clicks should bubble up to the onClick handler naturally. If marked adds target="_blank", override the link renderer:
   ```typescript
   renderer.link = ({ href, text }: { href: string; text: string }) => {
     return `<a href="${href}">${text}</a>`;
   };
   ```
   Add this to the `createCodeRenderer` function (rename it to `createCustomRenderer` or add the link override alongside the code override).
