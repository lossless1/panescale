# Quick Task 260319-l62: Auto-edit Text Files, Preview MD, Skip Binary - Summary

Editable textarea default for text file tiles with binary extension blocking and Cmd+S save

## What Was Done

### Task 1: Binary detection and canvas guard (5318653)
- Added `BINARY_EXTENSIONS` Set constant with 50+ common binary file extensions (archives, executables, media, fonts, databases, office docs)
- Changed `extensionToTileType` return type from `ContentTileType` to `ContentTileType | null`
- Binary extensions return `null` which is checked first before note/image/file-preview classification
- Added null guard in Canvas.tsx `onMouseUp` handler: binary files are silently ignored when dragged to canvas

### Task 2: Edit mode with save and preview toggle (71aecf8)
- FilePreviewNode now defaults to edit mode (`isEditing: true`) with an editable textarea
- Added `editContent` state separate from `content` (saved version) for tracking unsaved changes
- Added `dirty` indicator (accent-colored dot) next to filename when unsaved changes exist
- Added Edit/Preview toggle button in title bar (same style pattern as NoteNode)
- Added Cmd/Ctrl+S keyboard shortcut to save via `writeTextFile` from `@tauri-apps/plugin-fs`
- Preview mode highlights `editContent` so switching to preview shows current edits, not last save
- Textarea styled with monospace font, transparent background, spellcheck off, proper nopan/nodrag classes

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing functionality] Preview mode shows current edits**
- **Found during:** Task 2
- **Issue:** Original highlight effect used `content` (saved state) as dependency. When user edits and switches to preview, they would see stale highlighted content instead of their current edits.
- **Fix:** Changed highlight effect to depend on `editContent` instead of `content`
- **Files modified:** src/components/canvas/FilePreviewNode.tsx
- **Commit:** 71aecf8
